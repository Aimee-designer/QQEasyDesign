## 1. 产品介绍

### 1.1 平台概述

QQ宝藏 EasyDesign 是一个专业的在线图片设计平台，专为宣发内容制作而设计。平台支持两种主要设计模式：

- **公众号长图模式**：适用于微信公众号文章配图、朋友圈分享等场景
- **小红书多图模式**：专为小红书笔记发布优化的多页设计

### 1.2 核心特性

- ✨ **双模式设计**：一个平台，两种专业模式
- 🎨 **丰富的内容类型**：文字、图片、视频、动画、表情包等
- 📚 **完整的模板系统**：保存、加载、导入导出模板
- 🎯 **精准的导出控制**：多分辨率、分段切图、批量下载
- 🌙 **现代化界面**：支持日夜间主题切换
- 💾 **数据持久化**：本地存储，数据不丢失

### 1.3 适用场景

| 使用场景 | 推荐模式 | 输出规格 |
|---------|---------|----------|
| 微信公众号文章配图 | 公众号长图 | 375px宽长图 |
| 朋友圈营销海报 | 公众号长图 | 1125px高清图 |
| 小红书笔记发布 | 小红书多图 | 1242×1660px多页 |
| 产品宣传长图 | 公众号长图 | 支持分段切图 |
| 活动推广素材 | 两种模式均可 | 根据平台需求选择 |

## 2. 快速入门

### 2.1 第一次使用

1. **打开平台**：在浏览器中打开 `QQEasyDesign_v2.html`
2. **选择模式**：点击顶部的"公众号长图"或"小红书多图"按钮
3. **开始设计**：点击画布中的"+ 添加新段落"开始创作

### 2.2 基础操作流程

```
选择设计模式 → 添加内容段落 → 编辑文字和媒体 → 调整样式 → 预览效果 → 导出图片
```

### 2.3 5分钟快速上手

**步骤1：创建第一个段落**
- 点击画布底部的"+ 添加新段落"按钮
- 选择"模块标题"类型
- 输入标题文字，如"产品介绍"

**步骤2：添加正文内容**
- 再次点击"+ 添加新段落"
- 选择"正文标准"类型
- 输入产品描述文字

**步骤3：插入图片**
- 点击"+ 添加新段落"
- 选择"图片"类型
- 拖拽图片文件到上传区域或点击上传

**步骤4：导出图片**
- 点击右上角"导出长图"按钮
- 选择分辨率（推荐1125 @3x）
- 点击"导出全图"完成

## 4. 内容编辑功能

### 4.1 文字编辑

#### 4.1.1 文字类型

平台提供4种预设的文字类型：

| 类型 | 样式特点 | 字号 | 适用场景 |
|------|---------|------|----------|
| 模块标题 (subtitle) | 带装饰线的标题样式 | 18px | 章节标题、模块分隔 |
| 正文大字 (h1) | 粗体大号文字 | 20px | 重点内容、小标题 |
| 正文标准 (body) | 标准正文样式 | 16px | 主要内容、详细描述 |
| 注释 (note) | 小号辅助文字 | 14px | 备注信息、补充说明 |

#### 4.1.2 文字格式化

**加粗功能**
- 点击工具栏中的 `B` 按钮
- 快捷键：选中文字后点击加粗

**颜色设置**
- **QQ品牌蓝**：#0099FF，用于重点强调
- **警示红色**：用于重要提醒
- **默认黑色**：标准文字颜色

**高亮文字**
- 使用语法：`[需要高亮的文字]`
- 自动转换为黄色背景高亮效果
- 适用于关键词标注

#### 4.1.3 实时编辑

- **直接编辑**：点击文字即可直接修改
- **换行支持**：按Enter键换行
- **富文本**：支持混合格式文字

### 4.2 图片编辑

#### 4.2.1 支持的媒体格式

**图片格式**
- JPEG / JPG
- PNG（支持透明背景）
- SVG（矢量图形）
- GIF（动态图片）
- WebP（现代格式）

**视频格式**
- MP4
- WebM

**动画格式**
- Lottie JSON文件（矢量动画）

#### 4.2.2 图片上传方式

**方式1：拖拽上传**
1. 选择"图片"段落类型
2. 直接拖拽图片文件到上传区域
3. 系统自动识别格式并上传

**方式2：点击上传**
1. 点击上传区域的"点击上传"按钮
2. 在文件选择器中选择图片
3. 确认上传

**方式3：粘贴上传**
1. 复制图片到剪贴板
2. 在上传区域按 Ctrl+V (Windows) 或 Cmd+V (Mac)
3. 自动粘贴并上传

#### 4.2.3 图片调整

**宽度调节**
- 使用滑块调整图片宽度
- 范围：10% - 100%
- 保持宽高比例

**媒体信息**
- 自动显示媒体类型标签
- 显示文件大小信息
- 格式识别提示

## 5. 模板管理系统

### 5.1 模板系统概述

平台提供完整的模板管理功能，支持模板的创建、保存、加载和分享，帮助用户提高设计效率和保持设计一致性。

### 5.2 模板保存

#### 5.2.1 保存为普通模板

**操作步骤**
1. 完成设计内容
2. 点击顶部"模版管理"下拉菜单
3. 选择"保存模板"
4. 输入模板名称
5. 确认保存

**存储特点**
- 保存设计结构和样式
- 不包含具体的媒体文件
- 存储在浏览器本地存储中
- 加载速度快

#### 5.2.2 保存为预制模板

**操作步骤**
1. 在段落管理面板中
2. 点击段落右侧的"固定为预制"按钮
3. 系统自动保存为预制模板

**存储特点**
- 包含完整的媒体数据
- 存储在IndexedDB中
- 支持大文件存储
- 可完整还原设计

### 5.3 模板加载

#### 5.3.1 模板分类

**内置模板**
- 系统预设的专业模板
- 涵盖常见使用场景
- 定期更新和优化

**用户模板**
- 用户自己保存的模板
- 显示创建时间
- 支持重命名和删除

**预制模板**
- 包含媒体数据的完整模板
- 显示预览缩略图
- 一键完整加载

#### 5.3.2 加载操作

**步骤说明**
1. 点击"加载模板"按钮
2. 在弹窗中浏览模板
3. 点击模板卡片加载
4. 确认替换当前设计

**模板信息**
- **模板名称**：显示模板标题
- **创建时间**：显示保存日期
- **背景色预览**：显示模板主色调
- **操作按钮**：加载、导出、删除

### 5.4 模板导入导出

#### 5.4.1 导出模板

**导出功能**
- 在模板列表中，每个模板都有"导出"按钮
- 支持导出为 `.edtpl.json` 格式文件
- 包含完整的设计数据和媒体文件

**导出内容**
- 设计结构数据
- 样式配置信息
- 媒体文件的Base64数据
- 模板元信息

**使用场景**
- 模板备份和归档
- 团队间模板分享
- 跨设备模板同步

#### 5.4.2 导入模板

**支持格式**
- `.json`：标准JSON格式模板
- `.edtpl.json`：包含媒体数据的完整模板

**导入步骤**
1. 点击"导入模板文件"按钮
2. 选择本地模板文件
3. 系统自动解析和验证
4. 导入成功后添加到模板库

**错误处理**
- 格式验证：检查文件格式是否正确
- 数据完整性：验证模板数据完整性
- 错误提示：显示具体的错误信息

### 5.5 模板管理技巧

#### 5.5.1 模板命名规范

**推荐命名方式**
- 按用途分类：`产品介绍_v1`、`活动宣传_春节`
- 按风格分类：`简约风格_蓝色`、`科技风_深色`
- 按尺寸分类：`长图_公众号`、`多图_小红书`

#### 5.5.2 模板组织策略

**分类管理**
- 按项目分组保存模板
- 定期清理不用的模板
- 为重要模板做备份

**版本控制**
- 在模板名称中标注版本号
- 保留设计演进的关键版本
- 记录模板的使用场景

#### 5.5.3 团队协作

**模板分享流程**
1. 设计师创建标准模板
2. 导出为 `.edtpl.json` 文件
3. 通过文件分享给团队成员
4. 团队成员导入使用

**标准化建议**
- 制定团队模板命名规范
- 建立模板审核流程
- 定期更新和维护模板库

## 6. 导出功能详解

### 6.1 导出功能概述

平台提供灵活的导出功能，支持多种分辨率、格式和导出方式，满足不同平台和使用场景的需求。

### 6.2 公众号长图导出

#### 6.2.1 分辨率选择

**750px @2x**
- 输出宽度：750像素
- 适用场景：一般质量要求
- 文件大小：较小
- 加载速度：较快

**1125px @3x（推荐）**
- 输出宽度：1125像素
- 适用场景：高清显示需求
- 文件大小：较大
- 显示效果：更清晰

#### 6.2.2 导出方式

**完整长图导出**
- 点击"导出全图"按钮
- 生成一张完整的PNG图片
- 适合完整内容展示

**分段切图导出**
- **切2段**：将长图平均分为2部分
- **切3段**：将长图平均分为3部分
- **切4段**：将长图平均分为4部分
- **切5段**：将长图平均分为5部分

**分段导出优势**
- 适合社交媒体发布
- 减少单张图片文件大小
- 提高加载速度
- 便于分步骤展示内容

#### 6.2.3 批量下载

**自动打包**
- 分段导出时自动生成ZIP文件
- 包含所有切分的图片
- 文件命名规则：`设计标题_第X段.png`

**下载流程**
1. 选择导出方式和分辨率
2. 点击相应的导出按钮
3. 等待图片生成（显示进度）
4. 自动下载ZIP文件或单张图片

### 6.3 小红书多图导出

#### 6.3.1 输出规格

**固定尺寸**
- 宽度：1242像素
- 高度：1660像素
- 比例：3:4（小红书标准比例）
- 格式：PNG

#### 6.3.2 多页导出流程

**导出步骤**
1. 点击"导出小红书多图"按钮
2. 系统逐页生成图片
3. 显示导出进度
4. 自动依次下载所有页面

**文件命名**
- 格式：`设计标题_第X页.png`
- 按页面顺序编号
- 便于上传时保持顺序

#### 6.3.3 导出优化

**质量控制**
- 自动优化图片质量
- 保持文字清晰度
- 优化媒体元素显示

**兼容性**
- 符合小红书平台要求
- 支持透明背景
- 优化文件大小

### 6.4 导出技术细节

#### 6.4.1 技术实现

**html2canvas技术**
- 将DOM元素转换为Canvas
- 支持复杂样式渲染
- 保持高质量输出

**背景处理**
- 支持纯色背景导出
- 支持渐变背景导出
- 自动处理透明元素

#### 6.4.2 导出优化

**元素隐藏**
- 导出时自动隐藏编辑辅助元素
- 隐藏段落边框和操作按钮
- 保持纯净的视觉效果

**高DPI支持**
- 支持高分辨率屏幕
- 自动适配设备像素比
- 保证输出质量

#### 6.4.3 性能优化

**分批处理**
- 大型设计分批导出
- 避免浏览器内存溢出
- 提供进度反馈

**缓存机制**
- 缓存已处理的媒体元素
- 提高重复导出速度
- 减少计算资源消耗

### 6.5 导出最佳实践

#### 6.5.1 导出前检查

**内容检查清单**
- [ ] 所有文字内容已确认无误

## 8. 常见问题解答

### 8.1 使用问题

#### Q1: 为什么我的图片上传失败？
**A:** 可能的原因和解决方案：
- **文件格式不支持**：确保使用JPEG、PNG、GIF、SVG或WebP格式
- **文件过大**：建议单个文件不超过10MB
- **浏览器兼容性**：尝试使用Chrome、Firefox或Safari浏览器
- **网络问题**：检查网络连接，尝试重新上传

#### Q2: 导出的图片质量不清晰怎么办？
**A:** 优化建议：
- 选择更高的分辨率：使用1125px @3x而不是750px @2x
- 检查原始图片质量：确保上传的图片本身清晰
- 避免过度缩放：不要将小图片放大太多
- 使用PNG格式：对于有文字的图片，PNG比JPEG更清晰

#### Q3: 模板保存后找不到了？
**A:** 检查步骤：
- 确认保存操作：检查是否看到"保存成功"提示
- 查看正确分类：检查"用户模板"或"预制模板"分类
- 清除浏览器缓存：尝试刷新页面
- 检查浏览器存储：确保浏览器允许本地存储

#### Q4: 小红书多图模式下段落显示不全？
**A:** 解决方法：
- 检查段落分配：确保段落已正确分配到页面
- 调整内容长度：单页内容不要过多
- 使用分页功能：将内容合理分布到多个页面
- 检查元素可见性：确保元素没有被隐藏

### 8.2 技术问题

#### Q5: 浏览器提示内存不足？
**A:** 优化措施：
- 减少同时编辑的段落数量
- 压缩或删除不必要的大图片
- 关闭其他浏览器标签页
- 重启浏览器释放内存
- 分段设计复杂内容

#### Q6: 导出时卡住不动？
**A:** 排查步骤：
- 等待更长时间：复杂设计需要更多处理时间
- 检查浏览器控制台：查看是否有错误信息
- 尝试分段导出：使用分段切图功能
- 重新加载页面：刷新后重试导出
- 更换浏览器：尝试使用其他浏览器

#### Q7: 模板导入失败？
**A:** 检查项目：
- 文件格式正确：确保是.json或.edtpl.json格式
- 文件完整性：确保文件没有损坏
- 文件大小：检查文件是否过大
- JSON格式：使用文本编辑器检查JSON格式是否正确

### 8.3 设计问题

#### Q8: 如何保持设计风格一致？
**A:** 建议方法：
- 使用统一的色彩方案
- 建立标准模板库
- 制定字体使用规范
- 定期审核设计输出

#### Q9: 长图内容太多怎么办？
**A:** 处理策略：
- 使用分段切图导出
- 重新组织内容结构
- 删除不必要的信息
- 考虑制作多张图片

#### Q10: 如何提高设计效率？
**A:** 效率技巧：
- 建立常用模板库
- 使用快捷键操作
- 批量处理相似内容
- 制定标准工作流程

### 8.4 兼容性问题

#### Q11: 在某些浏览器上功能异常？
**A:** 兼容性建议：
- **推荐浏览器**：Chrome 80+、Firefox 75+、Safari 13+
- **避免使用**：IE浏览器、过旧版本浏览器
- **移动端**：iOS Safari、Android Chrome
- **更新浏览器**：保持浏览器为最新版本

#### Q12: 在不同设备上显示效果不同？
**A:** 适配说明：
- 编辑界面可能因屏幕尺寸有所差异
- 导出的图片在所有设备上显示一致
- 建议在桌面端进行设计工作
- 可在移动端预览最终效果

### 8.5 数据安全

#### Q13: 我的设计数据会丢失吗？
**A:** 数据保护：
- 数据存储在本地浏览器中
- 不会上传到服务器
- 建议定期导出模板备份
- 清除浏览器数据会删除设计

#### Q14: 如何备份我的设计？
**A:** 备份方法：
- 使用模板导出功能
- 定期保存重要设计为模板
- 将导出的.edtpl.json文件保存到云盘
- 建立团队共享的模板库
